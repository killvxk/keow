  • Fight Patents abuse by EC authorities - Gegen Patentmißbrauch durch die EU-Behörden.
  • syscall-no. in eax; args left to right ebx,ecx,edx,esi,edi(,ebp), all preserved.
  • returned 0xfffff000 < eax <= 0xffffffff may be an error code (-4096 < eax < 0).
  • Common reference:
    • <unistd.h>, <linux/types.h>, <asm/types.h>, <asm/posix_types.h>, and <linux/kernel.h> for data structures.
    • The syscall interface is defined in arch/i386/kernel/entry.S, symbols for loadable modules kernel/ksyms.c.
    • Error numbers and messages in include/asm/errno.h.
    • Basic data types
      short: word (16bit), int: dword (32bit), long: dword (32bit), pointer: dword (32bit).

  14.   newselect K4:select 142  
      watch i/o events and exceptions at selected filedescriptors w. timeout
      arg eax142 
      ebxuint top fd number + 1,up to FILE_MAX
      ecxzero or, ptr to fd bitfield for input;  
      edxzero or, ptr to fd bitfield for output;  
      esizero or, ptr to fd bitfield for exceptions;  
      edizero or, ptr to timeout values by timeval structure;  
      return eax no. of set fd-s or, zero after timeout.
          bitfield and timeval data may unpredictably be modified by syscall.
      errors eax badf: false fd bit set in a bitmask, intr: un-blocked signal received,
    inval: false top fd no., nomem: no memory for internal workspace
      ref arch/i386/kernel/sys_i386.c
    fdset files bitmask, timeval timeout data structure; oldselect
    a simple example, man 2 select_tut  
    NOTE:
    • ptr to an fdset bitfield
      • bit-position of fd number wrt the bitfields base address, set for being watched,
      • rsp mode will be ignored if ptr = zero.
      • read the FILE_MAX kernel constant, max. of open files, from "/proc/sys/fs/file-max"
    • ptr to timeval
      • zero timeout returns immediately,
      • infinite blocking if ptr to timeval structure = zero,
      • -ve timeout returns INVAL.

Notes

  • big endian vs little endian
    Gulliver's Travels: Those cutting their breakfast eggs at the big end and those who'd cut them at the little end were thus founding two entirely different ethnical groups, which lead to violent disputes about who was right. - A computer's big endian data order is where the most significant item comes 1st in address space, and consequently, little endian is where the less significant one is 1st, at least significant address unit.
